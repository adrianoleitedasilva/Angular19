***.vscode*** -> A pasta **`.vscode`** é usada para armazenar **configurações específicas do projeto no Visual Studio Code**. Ela normalmente contém arquivos como:
	- `settings.json` → configurações do editor para aquele projeto
	- `extensions.json` → extensões recomendadas
	- `launch.json` → configurações de debug
	- `tasks.json` → tarefas automatizadas
***node_mopdules*** -> A pasta **`node_modules`** é onde o **Node.js** armazena todas as dependências (bibliotecas e pacotes) que o seu projeto utiliza.
***public*** -> a pasta **`public`** (quando existe no projeto) é usada para armazenar **arquivos estáticos públicos**, que são copiados diretamente para a pasta de build e servidos sem processamento.
***src*** -> Ela contém todo o **código-fonte da aplicação**, ou seja, tudo que será compilado e transformado na aplicação final.
***.editorconfig*** -> O arquivo **`.editorconfig`** é um arquivo de configuração que padroniza regras de formatação de código dentro de um projeto.
***.gitignore*** -> O arquivo **`.gitignore`** serve para dizer ao Git **quais arquivos e pastas não devem ser versionados**. Ele evita que arquivos desnecessários, temporários ou sensíveis sejam enviados para o repositório.
***angular.json*** -> O **`angular.json`** é o arquivo que **configura como o Angular CLI constrói e executa o projeto**. Ele define regras de build, serve, testes e organização da aplicação.
***package.json*** -> O **`package.json`** é o arquivo principal de configuração de um projeto Node.js (e Angular também). Ele define:
- Nome e versão do projeto
- Dependências (`dependencies`)
- Dependências de desenvolvimento (`devDependencies`)
- Scripts (ex: `start`, `build`, `test`)
***package-lock.json*** -> O **`package-lock.json`** registra **a versão exata de cada dependência instalada**, incluindo dependências internas. Ele garante que:
- Todos do time instalem exatamente as mesmas versões
- O projeto funcione igual em qualquer máquina
- O ambiente de produção seja idêntico ao de desenvolvimento
***tsconfig.json*** -> É o **arquivo base de configuração do TypeScript**. Ele serve como configuração principal que os outros arquivos herdam. Define regras como:
- Versão do JavaScript gerado (`target`)
- Sistema de módulos
- Regras de tipagem
- Strict mode
- Paths e aliases
***tsconfig.app.json** -> Configuração específica **para a aplicação Angular**. Ele herda do `tsconfig.json`. Define:
- Quais arquivos fazem parte da aplicação
- Exclusões (como testes)
- Configurações específicas do build da app
***tsconfig.spec.json*** -> Configuração específica **para os testes** (arquivos `.spec.ts`). Ele inclui configurações próprias para ambiente de testes (Jasmine/Karma).

***style.scss*** -> O **`styles.scss`** é o arquivo de **estilos globais da aplicação Angular** usando **SASS/SCSS**. Ele define CSS que vale para **toda a aplicação**, diferente dos estilos específicos de cada componente. 
***main.ts*** -> O **`main.ts`** é o **ponto de entrada da aplicação Angular**. É o primeiro arquivo executado quando a aplicação inicia.
***index.html*** -> O **`index.html`** é a **página base da aplicação Angular**. É o único HTML “real” que o navegador carrega diretamente. A partir dele, o Angular assume o controle da aplicação.

![[Pasted image 20260302171938.png]]
***app.component.html*** -> É o **template HTML** do componente principal. Aqui fica a estrutura visual (layout, router-outlet, header, etc).
***app.component.scss*** -> Arquivo de **estilos específicos do AppComponent**. Os estilos aqui são isolados (encapsulados) e não afetam outros componentes.
***app.component.ts*** -> É o **arquivo principal do componente**. É onde o comportamento do componente é definido. Contém:
- Classe do componente
- Lógica
- Propriedades
- Métodos
- Decorator `@Component`
***app.component.spec.ts*** -> Arquivo de **teste unitário** do componente. Serve para validar se o componente funciona corretamente. Usado quando você executa: ng test
***app.config.ts*** -> Arquivo de **configuração da aplicação** (mais comum nas versões mais recentes do Angular com standalone components). Define:
- Providers globais
- Configuração de rotas
- Configuração do router
- Interceptors
***app.routes.ts*** -> Arquivo onde ficam definidas as **rotas da aplicação**.
