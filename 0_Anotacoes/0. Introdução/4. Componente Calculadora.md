import { Component } from '@angular/core';

  selector: 'app-calculadora',
  imports: [FormsModule],
  templateUrl: './calculadora.component.html',
  styleUrl: './calculadora.component.scss',
})

export class CalculadoraComponent {
  numero1: number = 0; // Poderiamos inserir uma interrogacao logo após o nome da var para tornar a declaracao do valor opcional.

  }
}

## No **Angular**, _data binding_ é o mecanismo que conecta **os dados do componente (TypeScript)** com **a interface (HTML template)**.  

Ele permite que **dados fluam entre a lógica da aplicação e a UI automaticamente**, mantendo tudo sincronizado.

Existem **4 formas principais de Data Binding no Angular**. Vou explicar cada uma de forma simples 👇

### Interpolação (Interpolation)

A **interpolação** usa `{{ }}` para mostrar valores do componente no HTML.

**Exemplo**

**Componente (TS)**

```typescript
export class AppComponent {
  nome = "Adriano";
}
```

**Template (HTML)**
```html
<h1>Olá {{ nome }}</h1>
```

### Property Binding

Usa **colchetes `[ ]`** para ligar uma propriedade do componente a uma **propriedade do elemento HTML**.

**Exemplo**

**Componente (TS)**

```typescript
export class AppComponent {
  imagemUrl = "foto.png";
}
```

**Template (HTML)**

```html
<img [src]="imagemUrl">
```

Outro exemplo:

```html
<button [disabled]="isDisabled">Salvar</button>
```

### Event Binding

Usa **parênteses `( )`** para capturar eventos do HTML e executar funções no componente.

**Exemplo**

**Componente (TS)**

```typescript
export class AppComponent {
  mostrarMensagem() {
    alert("Botão clicado!");
  }
}
```

**Template (HTML)**
```html
<button (click)="mostrarMensagem()">
  Clique aqui
</button>
```

### Two-Way Data Binding

O **Two-Way Binding** sincroniza **template e componente ao mesmo tempo**.

Sintaxe famosa chamada **"banana in a box"** 🍌📦: [(ngModel)]

**Exemplo**

**Componente (TS)**

```typescript
export class AppComponent {
  nome = "";
}
```

**Template (HTML)**
```html
<input [(ngModel)]="nome">

<p>Olá {{ nome }}</p>
```

📌 Quando o usuário digita, o valor muda no componente automaticamente.

#### ⚠ Para usar `ngModel` é necessário importar:

```typescript
import { FormsModule } from '@angular/forms';
```

## Resumo dos Data Bindings

|Tipo|Sintaxe|Fluxo|
|---|---|---|
|Interpolação|`{{ valor }}`|Component ➜ View|
|Property Binding|`[property]`|Component ➜ View|
|Event Binding|`(event)`|View ➜ Component|
|Two-way Binding|`[(ngModel)]`|Component ⇄ View|
#### Dica importante (nível profissional):

O **Two-Way Binding é pouco usado em aplicações grandes**.  
Na maioria dos projetos Angular modernos usa-se:
- **Reactive Forms**
- **Event + Property Binding**
- **State Management (NgRx)**
